# P1-02 — Viewer: Thumbnail Sidebar & Outline Navigation

**Epic:** E3 · **Primary package:** `Packages/DocumentSession` (sidebar views) · **Complexity:** M · **Priority:** High

**Owner:** claude-agent · **Branch:** task/P1-02-thumbnails-outline · **Claimed:** 822ecb9a7fd02cc0e47260ac22c16b8d1438bc3e

## Goal
Sidebar with page thumbnails (virtualized) and document outline/TOC tree; click-to-navigate; current-page tracking.

## Background
PRD FR-1.2. Thumbnails become the drag-reorder surface in P1-06 — build with that consumer in mind (selection model, item identity).

## Requirements
- Virtualized thumbnail list using low-res engine tiles; multi-select model.
- Outline tree from engine metadata (add outline read to DocEngineHost if missing — coordinate: this may touch `DocEngineHost`, mark PR `[INTEGRATION]` if so).
- Two-way sync: scroll updates sidebar highlight; sidebar click navigates.

## Dependencies
- P1-01.

## Files Likely Affected
- `Packages/DocumentSession/Sources/Sidebar/**`; possibly `Packages/DocEngineHost` (outline read).

## Acceptance Criteria
- 1,000-page fixture: sidebar opens instantly, memory bounded (virtualization verified).
- Outline navigation lands on correct destination incl. nested bookmarks with zoom targets.

## Definition of Done
- Global DoD.

## Testing Requirements
- Unit tests for selection model; snapshot tests; outline parsing tests against corpus manifest expectations.

## Documentation Updates
- None beyond package `CLAUDE.md` if selection model API added.

## Journal

- Engine seam: `OutlineReader` protocol + `OutlineNode` added to `PDFEngineAPI` (frozen seam) under **ADR-013**; `FakePDFEngine` gained `seedOutline`, `PDFEngineConformanceSuite.verifyOutlineReaderEmpty` added. `PDFiumEngine` implements it via `FPDFBookmark_*` (`fpdf_doc.h` added to `CPDFium`), with visited-set + depth-cap protection against cyclic/malformed bookmark trees.
- Fixture: no starter corpus PDF has a usable outline (uscis-i9's `/Outlines` is empty per PDFKit), so `Fixtures/pdf-corpus/synthetic/outlined-nested.pdf` was added — deterministically generated by `generate_outlined.py` in the same directory, manifest row `synthetic-outlined-nested` records the expected outline tree (nesting, XYZ zoom 1.5, no-dest structural heading). `DocEngineHostTests.testOutlineReaderParsesNestedBookmarksZoomAndUnlinkedHeading` pins it against real PDFium.
- Sidebar: `Sources/DocumentSession/Sidebar/` — `ThumbnailSelectionModel` (pure value type, click/⌘/⇧ semantics, P1-06 identity caveat documented on the type), `ThumbnailSidebarView` (LazyVStack-virtualized, one low-res full-page tile per row at scale 0.15 = shared cache entry with `PageTileView`'s placeholder), `OutlineSidebarView` (`OutlineGroup` tree, structural headings non-clickable), `DocumentSidebarView` (segmented Pages/Outline).
- Two-way sync: viewer reports `pageDidBecomeVisible` → `currentPage` highlights + auto-scrolls the sidebar; sidebar click / outline entry → `navigate(to:zoom:)` publishes a UUID-wrapped `NavigationTarget` the scroller consumes via `onChange` (UUID so repeat clicks on the same page still fire). Outline zoom targets apply before the scroll.
- Acceptance evidence: 1,000-page — `testOpeningAThousandPageDocumentDoesNoEagerPerPageWork` (open does zero renders/zero per-page metadata reads; row laziness + TileCache byte budget bound memory). Outline destinations incl. nested + zoom — the DocEngineHost fixture test above + `DocumentViewModelSidebarTests`. Selection model — `ThumbnailSelectionModelTests` (8 cases).
- Snapshot tests: deliberately not added — no snapshot-testing dependency exists in the approved set (CLAUDE.md §17 default-no); adding one is its own ADR-worthy decision, not a P1-02 drive-by. View-model unit tests cover the sync/selection/navigation contracts instead.
- App wiring: `AppDelegate` passes the same `PDFiumEngine` as `outlineReader:` (per ADR-013; App/ not owned by any other in-progress task).
